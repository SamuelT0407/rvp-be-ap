# 修 Bug 流程

## 概述

前端 Bug 通常以 UI 问题为主，例如样式错位、交互异常、数据显示错误等。

---

## 标准流程

### 1. 问题定位

- 确认 Bug 出现的**页面路径** (如 `/views/system/user/index.vue`)
- 确认 Bug 类型：
  - **UI 样式问题**: 样式错位、响应式异常
  - **数据问题**: 显示错误、字段缺失
  - **交互问题**: 按钮无响应、表单验证失效
  - **API 问题**: 请求失败、返回数据格式异常

### 2. 检查影响范围

- 该组件是否被其他页面复用？
- 该 API 是否被其他地方调用？
- 修改是否会影响全局样式/工具函数？

### 3. 修复

- 优先在**最小范围**内修复
- 避免为了修复一个小问题而改动全局配置
- 如果发现是通用组件的问题，需评估影响后再修改

### 4. 验证

- 刷新页面，确认问题已解决
- 检查相关页面是否受影响
- 如果涉及 API，使用开发者工具 Network 检查请求响应

---

## 常见问题与解决方案

> 此部分会随着开发逐步补充

### 样式问题

| 问题              | 原因                        | 解决方案                             |
| ----------------- | --------------------------- | ------------------------------------ |
| el-table 列宽异常 | 缺少 `width` 或 `min-width` | 给列设置明确的宽度                   |
| Dialog 内容超出   | 内容过长未处理              | 添加 `max-height` + `overflow: auto` |

### 交互问题

| 问题           | 原因                       | 解决方案                       |
| -------------- | -------------------------- | ------------------------------ |
| 按钮点击无响应 | 事件未绑定或权限不足       | 检查 `@click` 和 `v-has-permi` |
| 表单验证不触发 | `prop` 与 `v-model` 不匹配 | 确保 prop 名称一致             |

### 数据问题

| 问题             | 原因             | 解决方案                  |
| ---------------- | ---------------- | ------------------------- |
| 列表为空         | API 请求参数错误 | 检查 `queryParams`        |
| 编辑时数据不回显 | 字段映射错误     | 检查 `Object.assign` 映射 |
